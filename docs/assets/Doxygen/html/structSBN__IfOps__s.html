Detailed Description

This structure contains function pointers to interface-specific versions of the key SBN functions. Every interface module must have an equivalent structure that points to the approprate functions for that interface.

int(* SBN_IfOps_s::ReportModuleStatus) (SBN_ModuleStatusPacket_t *Buffer)

Reports the status of the module. The status can be in a module-specific format but must be no larger than SBN_MOD_STATUS_MSG_SZ bytes (as defined in sbn_platform_cfg.h). The status packet is passed in initialized (with message ID and size), the module fills it, and upon return the SBN application sends the message over the software bus.

Parameters
Buffer[out]Status packet to fill
Peer[in]Peer to report status
Returns
SBN_NOT_IMPLEMENTED if the module does not implement this function SBN_SUCCESS otherwise
int(* SBN_IfOps_s::Send) (SBN_PeerInterface_t *Peer, SBN_MsgType_t MsgType, SBN_MsgSz_t MsgSz, void *Payload)

Sends a message to a peer over the specified interface. Both protocol and data message buffers are included in the parameters, but only one is used at a time. The data message buffer is used for un/subscriptions and app messages. The protocol message buffer is used for announce and heartbeat messages/acks.

Parameters
Net[in]Interface data for the network where this peer lives.
Peer[in]Interface data describing the intended peer recipient.
MsgType[in]The SBN message type.
MsgSz[in]The size of the SBN message payload.
Payload[in]The SBN message payload.
Returns
Number of bytes sent on success, -1 on error.
